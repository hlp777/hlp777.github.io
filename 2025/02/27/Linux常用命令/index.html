Linux常用命令

一、系统信息与监控

1. uptime

查看系统运行时间及负载情况。

  • 输出说明：当前时间、运行时间、登录用户数、1/5/15分钟的平均负载（CPU 负载）。

2. top / htop

实时监控系统资源使用情况。

  • 关键操作：按 P（CPU 排序）、M（内存排序）、q 退出。

3. df / du

  • **df -h**：查看磁盘空间使用情况。
  • **du -sh /path**：统计目录大小。

二、文件与目录操作

4. ls

列出目录内容。

5. find

查找文件。

6. grep

文本搜索工具。

7. tar / gzip

压缩与解压文件。

三、网络管理

8. netstat / ss

查看网络连接状态。

9. curl / wget

网络请求与下载。

10. tcpdump

抓包分析网络流量。

四、进程管理

11. ps

查看进程状态。

12. kill

终止进程。

13. systemctl

管理系统服务。

五、权限管理

14. chmod

修改文件权限。

15. chown

修改文件所有者。

六、日志分析

16. tail / head

查看文件首尾内容。

17. journalctl

查看系统日志（Systemd 系统）。

七、软件包管理

18. APT/YUM/DNF

包管理工具（Debian/RedHat 系）。

八、高级工具

19. awk / sed

文本处理工具。

20. crontab

定时任务管理。

九、安全相关

21. sudo / su

权限切换。

总结

以上命令覆盖了运维工程师日常工作的核心场景，包括系统监控、文件操作、网络调试、服务管理等。建议结合以下技巧提升效率：

  • **管道符 |**：组合命令，如 cat file | grep "error"。
  • **通配符 * / ?**：快速匹配文件，如 rm *.tmp。
  • Tab 补全：减少输入错误。
  • 命令历史：Ctrl+R 搜索历史命令。

进阶使用

一、性能分析与调试

1. strace

追踪进程的系统调用（调试程序行为）。

2. lsof

列出系统打开的文件和网络连接。

3. sar

系统性能历史数据分析（需安装 sysstat）。

4. perf

性能分析工具（需安装 linux-tools-common）。

二、文件与存储高级操作

5. rsync

高效同步文件（支持增量、排除文件）。

6. dd

磁盘读写与备份（谨慎使用！）。

7. mount / umount

挂载与卸载文件系统。

8. ln

创建硬链接与软链接。

三、网络进阶工具

9. nmap

网络扫描与端口探测。

10. mtr

结合 traceroute 和 ping 的网络诊断工具。

11. iptables / nftables

防火墙规则管理。

12. nc (netcat)

网络调试瑞士军刀。

四、进程与资源控制

13. nohup

让进程在后台持久运行（忽略挂断信号）。

14. cgroups

限制进程资源使用（CPU/内存/磁盘等）。

15. taskset

绑定进程到特定CPU核心。

五、文本处理与脚本

16. jq

JSON数据处理工具。

17. xargs

将输入转换为命令参数。

18. cut / paste

按列切割或合并文本。

六、安全与审计

19. openssl

SSL/TLS证书管理。

20. auditctl

审计系统调用（需安装 auditd）。

七、容器与虚拟化

21. docker

容器管理（非原生Linux命令但广泛使用）。

22. virsh

管理KVM虚拟机。

八、高级技巧与组合命令

1. 快速分析日志

2. 批量重命名文件

3. 实时监控多个日志文件

4. SSH 隧道与端口转发

总结

掌握以上命令和技巧后，可应对以下场景：

  • 性能瓶颈分析：strace + perf + sar
  • 复杂文本处理：awk + jq + xargs
  • 网络故障排查：tcpdump + mtr + nmap
  • 自动化运维：crontab + rsync + 脚本组合
